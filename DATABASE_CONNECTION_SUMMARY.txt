🔧 SETUP METHODS

  METHOD 1: Environment Variables (Recommended)
  ─────────────────────────────────────────────
  
  Windows (PowerShell):
    $env:DB_HOST="ec2-xx-xxx-xxx-xxx.compute-1.amazonaws.com"
    $env:DB_USER="neondb_owner"
    $env:DB_PASS="your_password"
    $env:DB_NAME="neondb"
    $env:DB_PORT="5432"
  
  Windows (Command Prompt):
    set DB_HOST=ec2-xx-xxx-xxx-xxx.compute-1.amazonaws.com
    set DB_USER=neondb_owner
    set DB_PASS=your_password
    set DB_NAME=neondb
    set DB_PORT=5432
  
  Linux/Mac:
    export DB_HOST="ec2-xx-xxx-xxx-xxx.compute-1.amazonaws.com"
    export DB_USER="neondb_owner"
    export DB_PASS="your_password"
    export DB_NAME="neondb"
    export DB_PORT="5432"

  METHOD 2: .env File (Easiest)
  ─────────────────────────────
  
  1. Copy .env.example to .env
  2. Edit .env with your credentials
  3. Application automatically loads it
  
  Example .env:
    DB_HOST=ec2-xx-xxx-xxx-xxx.compute-1.amazonaws.com
    DB_USER=neondb_owner
    DB_PASS=your_password
    DB_NAME=neondb
    DB_PORT=5432
    RENDER=0

  METHOD 3: Direct Configuration
  ──────────────────────────────
  
  Edit config/database/db.php:
    define('DB_HOST', 'ec2-xx-xxx-xxx-xxx.compute-1.amazonaws.com');
    define('DB_USER', 'neondb_owner');
    define('DB_PASS', 'your_password');
    define('DB_NAME', 'neondb');
    define('DB_PORT', 5432);

🐛 TROUBLESHOOTING

  PROBLEM: Connection Refused
  ──────────────────────────
  Cause: Wrong host, port, or Neon service down
  Solution:
    1. Verify host from Neon console
    2. Ensure port is 5432
    3. Check internet connection

  PROBLEM: Authentication Failed
  ──────────────────────────────
  Cause: Wrong password or username
  Solution:
    1. Copy credentials from Neon console again
    2. Check for special characters
    3. Reset password in Neon if needed

  PROBLEM: Database Not Found
  ──────────────────────────
  Cause: Wrong database name or doesn't exist
  Solution:
    1. Verify database name from Neon console
    2. Create database if it doesn't exist

  PROBLEM: SSL Connection Error
  ────────────────────────────
  Cause: SSL not enabled or missing extension
  Solution:
    1. Ensure sslmode=require is set
    2. Update PHP to latest version
    3. Install php-pgsql extension

  PROBLEM: Timeout Error
  ────────────────────
  Cause: Slow connection or firewall blocking
  Solution:
    1. Check internet connection
    2. Check firewall settings
    3. Try again later
