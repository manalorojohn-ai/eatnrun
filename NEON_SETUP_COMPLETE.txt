🚀 GETTING STARTED (3 STEPS)

  STEP 1: Get Your Neon Connection String
  ────────────────────────────────────────
  
  1. Go to https://console.neon.tech
  2. Select your project
  3. Click "Connection string" or "Connect"
  4. Copy the connection string
  
  Example:
    postgresql://neondb_owner:password@ec2-xx-xxx-xxx-xxx.compute-1.amazonaws.com:5432/neondb?sslmode=require

  STEP 2: Create .env File
  ────────────────────────
  
  1. Copy .env.example to .env
  2. Edit .env with your Neon credentials
  
  Windows:
    copy .env.example .env
    notepad .env
  
  Linux/Mac:
    cp .env.example .env
    nano .env
  
  Add your credentials:
    DB_HOST=ec2-xx-xxx-xxx-xxx.compute-1.amazonaws.com
    DB_USER=neondb_owner
    DB_PASS=your_password
    DB_NAME=neondb
    DB_PORT=5432

  STEP 3: Test Connection
  ──────────────────────
  
  Option A - Web Interface:
    1. Open http://localhost/test_db_connection.php
    2. Enter your credentials
    3. Click "Test Connection"
  
  Option B - Command Line:
    php test_db_connection.php
  
  Option C - Interactive Setup:
    php setup_neon_connection.php setup

🔗 CONNECTION DETAILS EXTRACTION

  ────────────────────────────
  
  postgresql://[USER]:[PASSWORD]@[HOST]:[PORT]/[DATABASE]?sslmode=require

⚙️ CONFIGURATION METHODS

  METHOD 1: Environment Variables (Recommended)
  ──────────────────────────────────────────────
  
  Windows PowerShell:
    $env:DB_HOST="ec2-xx-xxx-xxx-xxx.compute-1.amazonaws.com"
    $env:DB_USER="neondb_owner"
    $env:DB_PASS="your_password"
    $env:DB_NAME="neondb"
    $env:DB_PORT="5432"
  
  Linux/Mac:
    export DB_HOST="ec2-xx-xxx-xxx-xxx.compute-1.amazonaws.com"
    export DB_USER="neondb_owner"
    export DB_PASS="your_password"
    export DB_NAME="neondb"
    export DB_PORT="5432"

  METHOD 2: .env File (Easiest)
  ──────────────────────────────
  
  1. Copy .env.example to .env
  2. Edit with your credentials
  3. Application loads automatically
  
  Content:
    DB_HOST=ec2-xx-xxx-xxx-xxx.compute-1.amazonaws.com
    DB_USER=neondb_owner
    DB_PASS=your_password
    DB_NAME=neondb
    DB_PORT=5432
    RENDER=0

  METHOD 3: Direct Configuration
  ───────────────────────────────
  
  Edit config/database/db.php:
    define('DB_HOST', 'ec2-xx-xxx-xxx-xxx.compute-1.amazonaws.com');
    define('DB_USER', 'neondb_owner');
    define('DB_PASS', 'your_password');
    define('DB_NAME', 'neondb');
    define('DB_PORT', 5432);

🐛 QUICK TROUBLESHOOTING

  Connection Refused
  ──────────────────
  ✗ Error: Connection refused
  ✓ Solution: Verify host, port, and internet connection
  
  Authentication Failed
  ─────────────────────
  ✗ Error: password authentication failed
  ✓ Solution: Check username and password from Neon console
  
  Database Not Found
  ──────────────────
  ✗ Error: database does not exist
  ✓ Solution: Verify database name or create it in Neon
  
  SSL Connection Error
  ────────────────────
  ✗ Error: SSL connection error
  ✓ Solution: Ensure sslmode=require is set
  
  Timeout Error
  ─────────────
  ✗ Error: Connection timeout
  ✓ Solution: Check internet connection and firewall

🚀 DEPLOYMENT

  On Render.com
  ─────────────

  2. Go to "Environment"
  3. Add environment variables:
     DB_HOST, DB_USER, DB_PASS, DB_NAME, DB_PORT, RENDER=1

  ────────
  1. Go to your Heroku app
  2. Go to "Settings" → "Reveal Config Vars"
  3. Add environment variables
  4. Deploy

  On Other Platforms
  ──────────────────
  1. Set environment variables in hosting control panel
  2. Or use .env file (not in version control)
  3. Or modify config/database/db.php directly
